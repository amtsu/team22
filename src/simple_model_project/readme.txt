    моделирующая система пользуется отдельными модулями, обеспечивая
процесс моделирования каждого из них. Модули представляют собой файлы на языке python, в 
каждом из которых должно находиться описание класса, наследуемого от класса AbstractModule, 
описанного в файле abstractmodule.py, и функция создания экземпляра данного класса create_instance.
    Состав модуля:
	Функция create_instance не должна принимать никаких параметров, и должна после вызова 
возвращать экземпляр(объект) заданного в модуле класса.
    На конструктор класса не накладывается дополнительных ограничений, так как моделирующая система 
непосредственно не создает объекты, а использует для их создания функции create_instance.
    Метод name() не должен принимать никаких параметров, при этом должен возвращать строку 
с содержимым, достаточным для идентификации объекта класса среди других.
    Метод prepare() не принимает и не возвращает никаких значений,вызывается один раз 
перед началом цикла моделирования и предназначен для задания начального состояния объекта класса.
    Методы step() и is_done() предназначены для организации процесса моделирования. Метод step()
не принимает и не возвращает значений. Каждый вызов метода step() сигнализирует объекту 
о необходимости изменения состояния объекта на один шаг. Метод is_done() не должен принимать никаких 
значений и возвращать значение True в случае, когда объект достиг своего конечного состояния.
Если объект находится в промежуточном состоянии, метод is_done должен вернуть False.
    Метод get_state() не должен принимать никаких значений, но должен возвращать значение 
переменных, описывающих состояние модуля на текущем шаге в виде словаря 
    {имя переменной:значение}, 
например
        {"скорость, м/c" : "15", "пройденный путь, м":"12563"}
    Текущий функционал моделирующей системы:
Моделирующая система:
1. Анализирует каталог src\simple_model_project\ на предмет наличия модулей,
при анализе модулем считается любой py-файл, содержащий в себе метод create_instance().
2. Выполняет создание объектов классов для всех найденных модулей.
3. Выполняет вызов метода prepare() для каждого созданного объекта.
4. Выполняет вызов метода is_done() для каждого созданного объекта; если 
   данный метод возвращает False, то вызывается метод step() данного объекта.
5. После вызова step() для каждого созданного объекта осуществляется вызов метода 
   get_state(). Возвращаемые значения регистрируются ("запоминаются").   
6. п.4 и 5 повторяется пока для каждого объекта метод is_done() не начнет возвращать True, но не более 200 раз
7. После окончания выполнения действий п.4-6 зарегистрированные данные отображаются в графическом
   и табличном виде.
Процесс, описываемый п. 3 - 6 называется процессом моделирования.
